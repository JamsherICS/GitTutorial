Static Memory Allocation:

  • Static memory allocation is done at compile time.
  • Memory is allocated for variables when the program starts and remains allocated throughout the program's execution.
  • The size of the memory block is fixed and determined by the compiler.
  • Variables declared with static allocation are allocated on the stack or in the global memory area, depending on their scope.
  • Examples include global variables, local static variables, and arrays with fixed sizes.
  int global_variable; // Global variable
  static int static_variable; // Static variable
  void function() {
      static int local_static_variable; // Local static variable
  }

Dynamic Memory Allocation:

  • Dynamic memory allocation is done at runtime using library functions like malloc(), calloc(), and realloc() from the <stdlib.h> header.
  • Memory is allocated on the heap, a region of memory separate from the stack.
  • Memory allocation and deallocation can be done dynamically during program execution.
  • The size of the memory block can be determined at runtime, allowing for flexibility in memory management.
  • Dynamic memory allocation is useful when the size of data structures is not known at compile time or when memory needs to be allocated and deallocated dynamically.
  int global_variable; // Global variable
  static int static_variable; // Static variable
  void function() {
      static int local_static_variable; // Local static variable
  }
