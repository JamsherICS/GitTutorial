The malloc() and calloc() functions are used for dynamic memory allocation in C. However, they differ in their behavior:

  • malloc(): Allocates a block of memory of a specified size in bytes. The allocated memory is uninitialized, meaning it contains garbage values.
  • calloc(): Allocates a block of memory for an array of elements, initializes all bytes to zero, and returns a pointer to the allocated memory.

Here's a comparison between malloc() and calloc():

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr_malloc, *ptr_calloc;

    // Allocate memory using malloc()
    ptr_malloc = (int *)malloc(5 * sizeof(int));

    // Allocate memory using calloc()
    ptr_calloc = (int *)calloc(5, sizeof(int));

    // Print the memory addresses
    printf("Memory address using malloc(): %p\n", (void *)ptr_malloc);
    printf("Memory address using calloc(): %p\n", (void *)ptr_calloc);

    // Free allocated memory
    free(ptr_malloc);
    free(ptr_calloc);

    return 0;
}

In this example, malloc() and calloc() are used to allocate memory for 5 integers each. The difference is that memory allocated by malloc() contains garbage values, while memory allocated by calloc() is initialized to zero.
