In C, storage classes define the scope, visibility, and lifetime of variables. There are four storage classes in C:

  • Automatic (auto):
    • Variables declared inside a block without any storage class specifier are by default automatic.
    • They are created when the block is entered and destroyed when the block is exited.
    • They have local scope and are inaccessible outside the block where they are declared.
    • Example:
    void function() {
    auto int x; // Automatic variable
}

  • Register (register):
    • Variables declared with the register specifier are stored in CPU registers for faster access.
    • They have local scope and are inaccessible outside the block where they are declared.
    • Example:
    void function() {
    register int x; // Register variable
}

  • Static (static):
    • Static variables retain their value between function calls.
    • They have local scope within the block where they are declared but retain their value across function calls.
    • If declared at the global level, they are accessible only within the file where they are declared.
    • Example:
    void function() {
    static int x; // Static variable
}

  • External (extern):
    • External variables are declared outside all functions.
    • They can be accessed by any function within the file where they are declared or by functions in other files.
    • They have global scope.
    • Example:
    extern int x; // External variable declaration
