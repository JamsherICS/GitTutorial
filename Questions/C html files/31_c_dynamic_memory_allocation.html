Dynamic memory allocation in C refers to the process of allocating memory during program execution, allowing the program to request memory dynamically from the operating system at runtime. This enables the creation of data structures such as arrays and structures whose size is determined during runtime, rather than at compile time.

The standard library functions malloc(), calloc(), and realloc() are used for dynamic memory allocation in C:

  • malloc(): Allocates a block of memory of a specified size in bytes.
  • calloc(): Allocates a block of memory for an array of elements, initializes all bytes to zero, and returns a pointer to the allocated memory.
  • realloc(): Resizes the memory block pointed to by a previously allocated pointer, possibly moving it to a new location. It also allows for shrinking or expanding the memory block.

Here's an example of using malloc() to dynamically allocate memory for an integer:

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;

    // Allocate memory for an integer
    ptr = (int *)malloc(sizeof(int));

    if (ptr == NULL) {
        printf("Memory allocation failed\n");
        return 1;
    }

    *ptr = 10;
    printf("Value stored at ptr: %d\n", *ptr);

    // Deallocate memory
    free(ptr);

    return 0;
}

In this example, malloc() is used to allocate memory for an integer, and the allocated memory is then used to store the value 10. After using the memory, it is deallocated using the free() function to prevent memory leaks.
