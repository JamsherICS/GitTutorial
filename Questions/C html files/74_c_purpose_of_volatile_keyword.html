In C, the volatile keyword is used to indicate that a variable may be changed at any time by external factors that are beyond the control or prediction of the compiler. When you declare a variable as volatile, you're informing the compiler that its value can be changed unexpectedly, and therefore the compiler should not optimize or make assumptions about the variable's value or behavior.

The primary purpose of using the volatile keyword is to prevent the compiler from performing certain optimizations that could lead to incorrect behavior when dealing with memory-mapped hardware registers, signal handlers, or variables accessed by multiple threads in a concurrent program.

Example: 

  #include <stdio.h>
  
  volatile int flag = 0; // Declaring a volatile variable
  
  void toggle_flag() {
      flag = !flag; // Modify the volatile variable
  }
  
  int main() {
      while (1) {
          toggle_flag(); // Function call that changes the value of the volatile variable
          printf("Flag: %d\n", flag); // Access the volatile variable
      }
      return 0;
  }

Output:

  Flag: 1
  Flag: 0
  Flag: 1
  Flag: 0
  Flag: 1
  ...
