Error handling in C programs involves identifying and responding to unexpected or erroneous conditions that may occur during program execution. Here are common techniques for implementing error handling in C:

Return Values:

Functions can return error codes or special values to indicate errors. The calling code can check these return values and handle errors accordingly.

int result = someFunction();
      if (result < 0) {
          // Handle error
      }

Global Error Indicator:

Use a global variable to store error information. Functions can update this variable when errors occur, and the calling code can check it.

int error = 0;
      
      void someFunction() {
          // Error occurs
          error = 1;
      }
      
      int main() {
          someFunction();
          if (error) {
              // Handle error
          }
          return 0;
      }

Error Codes:

Define error codes for different types of errors using macros or enumeration constants. Functions can return these error codes, and the calling code can switch on them to determine the appropriate error handling.

#define ERROR_OUT_OF_MEMORY -1
      #define ERROR_FILE_NOT_FOUND -2
      
      int someFunction() {
          // Error occurs
          return ERROR_OUT_OF_MEMORY;
      }
      
      int main() {
          int result = someFunction();
          switch (result) {
              case ERROR_OUT_OF_MEMORY:
                  // Handle out of memory error
                  break;
              case ERROR_FILE_NOT_FOUND:
                  // Handle file not found error
                  break;
              default:
                  // Handle other errors
                  break;
          }
          return 0;
      }

Error Reporting Functions

Define functions for reporting errors, such as printing error messages to the console, logging errors to a file, or displaying error dialogs.

void reportError(const char *message) {
              fprintf(stderr, "Error: %s\n", message);
          }
          
          int main() {
              // Error occurs
              reportError("An error occurred");
              return 1;
          }

errno and perror

Use the errno variable and perror() function provided by the standard library to report errors related to system calls.

#include <stdio.h>
          #include <errno.h>
          
          int main() {
              FILE *file = fopen("nonexistent_file.txt", "r");
              if (file == NULL) {
                  perror("Error opening file");
                  printf("Error code: %d\n", errno);
              }
              return 0;
          }

Assertions

Use assertions (assert()) to check for conditions that should always be true during program execution. If an assertion fails, it indicates a bug in the program.

#include <assert.h>
          
          int main() {
              int x = 10;
              assert(x > 0);
              return 0;
          }

Choose the appropriate error handling technique based on the nature of the error and the requirements of your program. Effective error handling can improve the reliability and robustness of your C programs.
