PEP 8 - Python Enhancement Proposal 8

PEP 8 stands for Python Enhancement Proposal 8. It is a style guide for Python code, outlining best practices and conventions for writing clean, readable, and maintainable code. PEP 8 was authored by Guido van Rossum, Barry Warsaw, and Nick Coghlan, and it's considered the de facto standard for Python code style.

Key Points Covered by PEP 8:

  • Indentation: Use 4 spaces per indentation level. Never mix tabs and spaces.

  • Whitespace: Use spaces around operators and after commas, but not directly inside parentheses, brackets, or braces.

  • Naming Conventions: Use descriptive names for variables, functions, and classes. Use lowercase for variable names, uppercase for constants, and capitalize words in function and class names.

  • Comments: Write comments to explain non-obvious code. Keep them concise and use proper punctuation.
